# Mini Project Case：个人任务看板

这是一个示例案例，用来说明如何将 Agentic Development Workflow 应用于一个小型项目。

## 1. 原始需求

我想做一个简单的个人任务看板，可以添加任务、标记完成、按状态筛选，并保存到本地浏览器。

## 2. Clarify 输出

### 目标用户

个人开发者或学生，用于管理每日任务。

### 核心功能

- 添加任务；
- 标记完成；
- 删除任务；
- 按全部、未完成、已完成筛选；
- 使用 localStorage 保存数据。

### 非目标范围

- 不做登录；
- 不做云同步；
- 不做多人协作；
- 不做复杂日历视图。

### 验收标准

- 刷新页面后任务仍存在；
- 可以切换任务状态；
- 可以按状态筛选；
- 空任务不能提交；
- 页面在手机宽度下可使用。

## 3. Plan 输出

| 任务 | 目标 | 验收标准 |
| --- | --- | --- |
| 页面结构 | 创建输入框、任务列表和筛选按钮 | 页面元素完整 |
| 状态管理 | 管理任务数组和筛选状态 | 添加、完成、删除可用 |
| 本地存储 | 接入 localStorage | 刷新后数据不丢失 |
| 样式优化 | 适配桌面和移动端 | 小屏可读可点 |
| 文档整理 | 编写 README 和使用说明 | 新用户能运行项目 |

## 4. Build 示例提示

```text
请实现一个纯前端任务看板，使用 HTML、CSS、JavaScript，不引入框架。
功能包括：添加任务、标记完成、删除任务、按状态筛选、localStorage 保存。
请保持代码简单，并添加必要注释。
```

## 5. Debug 示例

### 问题

刷新页面后任务丢失。

### Agent 分析方向

- 是否在添加任务后写入 localStorage；
- 页面加载时是否读取 localStorage；
- JSON.parse 是否处理了空值；
- 是否每次渲染时覆盖了原始数据。

### 修复验证

1. 添加 2 个任务；
2. 刷新页面；
3. 确认任务仍存在；
4. 删除 1 个任务；
5. 再次刷新确认状态正确。

## 6. Review 重点

- 不要把任务数据上传到服务器；
- localStorage 读取失败时要有兜底；
- 删除操作要只影响目标任务；
- 空输入不能生成任务。

## 7. Ship 内容

- README；
- 在线演示地址；
- 功能截图；
- 已知限制说明。
