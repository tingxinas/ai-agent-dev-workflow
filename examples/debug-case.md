# Debug 案例示例

## 问题描述

在开发一个前端页面时，页面运行后出现组件状态没有及时更新的问题，表现为点击按钮后界面数据不刷新。

## 传统排查方式

1. 手动阅读组件代码；
2. 检查状态变量定义；
3. 检查事件绑定；
4. 检查异步更新逻辑；
5. 逐步添加日志定位问题。

## Agent 辅助排查方式

将报错信息、相关代码片段和现象描述交给 AI Agent，由 Agent 协助分析可能原因：

- 状态更新是否使用了旧闭包；
- 是否直接修改了对象引用；
- 是否缺少依赖项；
- 是否异步请求未正确处理；
- 是否组件没有重新渲染。

## 修复建议示例

Agent 可能给出的建议包括：

1. 避免直接修改 state 对象；
2. 使用函数式状态更新；
3. 检查 useEffect 依赖项；
4. 将异步请求结果统一写入状态；
5. 添加最小复现用例验证修复结果。

## 效果

通过 Agent 辅助，可以更快列出可能原因和排查路径，减少无方向阅读代码的时间。实际修复仍需要开发者结合项目上下文进行判断和验证。
