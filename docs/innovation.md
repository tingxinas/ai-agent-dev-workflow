# Innovation Notes

本项目的创新点不在于简单调用某个大模型，而在于把 AI Agent 融入软件开发流程，并为人机协作设计可复用的结构、边界和检查机制。

## 1. Agent Fit Score

很多 AI 开发流程默认“所有任务都可以交给 Agent”。本项目引入 Agent Fit Score，用来判断一个任务是否适合 AI 辅助。

评估维度包括：

- 是否是结构化、重复性任务；
- 是否可以小步验证；
- 是否涉及隐私、权限、支付或不可逆操作；
- 是否有明确验收标准；
- 是否需要复杂业务判断。

目标是避免盲目自动化，让 Agent 在适合的位置发挥作用。

## 2. Human-in-the-loop Gates

项目将人工审核设计成流程中的固定关卡，而不是事后补救。

关键关卡包括：

- 需求澄清后确认；
- 任务计划确认；
- 代码合并前审查；
- 发布前安全检查；
- 复盘和文档沉淀。

这让 AI 输出可以被持续验证，而不是一次性全自动执行。

## 3. Prompt as Artifact

项目把 Prompt 当成可维护资产，而不是一次性聊天内容。

每类 Prompt 都对应明确场景：

- 需求澄清；
- 任务拆解；
- 代码实现；
- Debug；
- Review；
- Ship。

这样可以让团队逐步沉淀适合自己的提示词库。

## 4. Release Guard

AI Agent 很容易生成看起来完整但不适合公开发布的内容。本项目加入 Release Guard，专门检查：

- 敏感信息；
- 截图隐私；
- Git 历史风险；
- 文档真实性；
- License 和依赖。

这让工作流不仅关注“能不能做出来”，也关注“能不能安全公开”。

## 5. Local-first Toolkit

`tools/index.html` 是一个本地优先的网页工具，不依赖后端，也不上传用户输入。

它适合处理早期项目想法、内部需求和调试信息，因为这些内容可能包含未公开信息。

## 6. Workflow Memory

项目鼓励把关键过程沉淀为 Markdown：

- 需求澄清记录；
- 任务计划；
- Debug 分析；
- 审查意见；
- 发布检查；
- 复盘总结。

这些记录可以形成项目级记忆，让 Agent 下一次更容易理解上下文。

## 7. 可扩展方向

未来可以继续扩展：

- 增加项目复杂度评分；
- 增加风险关键词自定义；
- 增加 Prompt 模板导入导出；
- 增加 GitHub Issue 模板生成；
- 增加项目复盘报告生成；
- 增加多 Agent 角色分工模拟。
