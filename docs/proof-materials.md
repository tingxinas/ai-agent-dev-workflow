# Evidence Notes

本文件用于记录 Agentic Development Workflow 的使用证据和过程材料。它既可以帮助团队复盘，也可以在需要展示 AI 辅助开发能力时作为材料索引。

## 1. 建议保留的过程记录

### 需求阶段

- 原始需求文本；
- Agent 澄清后的需求说明；
- 功能边界和验收标准；
- 关键取舍记录。

### 开发阶段

- Agent 生成的任务计划；
- 代码修改前后的对比；
- 关键模块实现说明；
- 人工审核意见。

### Debug 阶段

- 问题现象；
- 复现步骤；
- 报错日志；
- Agent 分析结果；
- 最终修复方式。

### 交付阶段

- README；
- 部署说明；
- 测试清单；
- 版本变更记录；
- 项目截图或演示地址。

## 2. 推荐文件命名

```text
01-requirement-clarification.md
02-agent-task-plan.md
03-code-review-notes.md
04-debug-analysis.md
05-release-checklist.md
06-demo-screenshot.png
```

## 3. 截图注意事项

公开截图前请确认：

- 没有账号、邮箱、手机号；
- 没有 Token、密钥、Cookie；
- 没有客户数据或内部业务信息；
- 没有无法公开的聊天记录；
- 浏览器地址栏和终端路径不会泄露隐私。

## 4. 可公开描述模板

```text
该项目记录了一套 AI Agent 辅助开发流程，覆盖需求澄清、任务拆解、代码实现、Debug 分析、人工审核和文档交付。项目中的文档和示例用于说明如何把 AI 工具系统性地融入软件开发流程，并强调所有 AI 输出都需要经过人工验证。
```

## 5. 不建议公开的内容

- 私有业务代码；
- 客户数据；
- 内部接口地址；
- 真实生产日志；
- 个人隐私截图；
- 未经授权的第三方材料。
