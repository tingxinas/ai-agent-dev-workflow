# 使用证明与影响力证明材料清单

本文件用于整理 AI Agent 辅助开发的证明材料，适合提交给 AI 工具申请、试用、资助或评审活动。

## 一、建议提交的证明材料

### 1. 工具使用截图

建议包含：

- Claude Code / Cursor / Codex 的使用界面；
- Agent 读取项目文件并生成修改方案的截图；
- AI 根据报错日志定位问题的截图；
- AI 生成 README、测试清单或部署说明的截图。

### 2. GitHub 项目链接

建议提供当前项目仓库链接，并确保仓库中包含：

- README.md；
- docs 工作流说明；
- examples 示例材料；
- demo 演示页面；
- 真实提交记录。

### 3. 项目运行效果

可以上传：

- 页面截图；
- 终端运行截图；
- 功能演示录屏；
- 部署后的在线访问地址。

### 4. AI 参与开发记录

可以整理以下内容：

- 需求拆解记录；
- Agent 任务计划；
- Debug 过程记录；
- 代码重构前后对比；
- 文档生成过程。

## 二、申请表单可填写内容

> 附件中包含 AI 开发工具使用截图、Agent 辅助生成代码与调试过程截图、项目运行效果截图，以及 GitHub 项目链接。相关材料可以证明本人长期使用 AI Agent 参与需求分析、代码生成、Bug 修复、测试验证和文档整理等开发流程，并已在实际项目中形成稳定工作流。

## 三、推荐附件命名

```text
01-ai-tool-usage.png
02-agent-task-plan.png
03-code-generation.png
04-debug-process.png
05-project-demo.png
06-github-commit-history.png
```

## 四、注意事项

- 不要上传包含账号密码、Token、密钥或隐私信息的截图；
- 尽量使用真实项目截图，而不是空白演示；
- 证明材料要能对应表单中填写的项目成果；
- 如果使用效率提升数据，建议说明为“个人项目经验估算”。
