# Public Release Checklist

公开发布 GitHub 仓库前，建议逐项检查，避免泄露隐私或上传不适合公开的内容。

## 1. 敏感信息检查

确认仓库中不包含：

- API Token；
- GitHub Token；
- 数据库密码；
- `.env` 文件；
- 私有服务器地址；
- 个人身份证、手机号、邮箱等隐私信息；
- 客户数据或业务机密；
- 未脱敏日志。

建议搜索关键词：

```text
password
secret
token
api_key
apikey
Authorization
Bearer
.env
```

## 2. 截图和媒体文件检查

如果上传截图、录屏或图片，确认：

- 浏览器地址栏没有敏感参数；
- 终端没有暴露用户名、服务器 IP 或密钥；
- 页面中没有个人信息；
- 聊天记录或客户数据已打码；
- 图片文件名不包含隐私信息。

## 3. Git 历史检查

即使当前文件已删除，敏感信息也可能存在于 Git 历史中。公开前建议确认：

- 首次提交前没有加入敏感文件；
- 如果误提交过敏感信息，需要重置历史或更换密钥；
- 不要依赖 `.gitignore` 清除已经提交过的内容。

## 4. 文档真实性检查

README 和文档中避免：

- 夸大效果；
- 使用无法证明的数字；
- 声称项目已大规模商用但没有依据；
- 混淆示例项目和生产项目；
- 使用第三方 Logo 或名称造成背书误解。

## 5. License 检查

确认：

- 仓库包含 License；
- 使用第三方代码时遵守对应许可证；
- 不复制未授权的商业代码、图片或模板。

## 6. 最小发布命令

```bash
git status
git log --oneline -5
git remote -v
git push -u origin main
```

## 7. 发布后检查

发布后打开 GitHub 页面确认：

- README 渲染正常；
- Mermaid 图表显示正常；
- 链接可以打开；
- 没有无关本地文件；
- 仓库描述清晰。
